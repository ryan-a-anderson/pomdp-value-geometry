# Value Function Geometry for Memoryless POMDPs

## Project Overview

This project explores the geometric structure of feasible value functions for **memoryless POMDP policies** and analyzes which policies are optimal for different initial state distributions.

**Key Question**: For a memoryless POMDP policy π: O → Δ_A (mapping observations to action distributions), which extreme point policies are optimal for different initial state distributions?

## Theoretical Background

### Memoryless POMDP Policies

Unlike belief-based POMDPs that maintain state distributions, we consider **memoryless policies**:
- Policy: π(a|o) - probability of action a given observation o
- No belief state tracking
- State-value function v: S → ℝ satisfies Bellman equation

### Parametric Linear System

For binary actions (a ∈ {0,1}), we can parameterize policies by:
- p_o = π(a=1|o) ∈ [0,1] for each observation o

The value function v satisfies:
```
A(p) v = b(p)
```

where A(p) and b(p) are affine in policy parameters:
```
A(p) = A₀ + Σ_o p_o A^o
b(p) = b₀ + Σ_o p_o b^o
```

### Midpoint-Radius Reparametrization

To analyze the feasible set, we use:
- **Midpoint**: p^c = [0.5, 0.5, ...] (center of parameter space)
- **Radius**: p^Δ = [0.5, 0.5, ...] (maximum deviation)

Define:
```
R^c(x) = A(p^c) x - b(p^c)                  (residual at center)
D_k(x) = p^Δ_k (A^k x - b^k)                (deviation generators)
D(x) = [D_1(x), D_2(x), ..., D_K(x)]        (deviation matrix)
```

### Feasibility via Zonotope Membership

A point x is a feasible value function iff:
```
R^c(x) ∈ zonotope generated by columns of D(x)
```

Equivalently, there exists q ∈ [-1,1]^K such that:
```
D(x) q = R^c(x)
```

### Facet-Defining Inequalities (2D)

In 2D, zonotope facets are defined by perpendicular vectors to generators:
- For generator D_i(x), take normal c ⊥ D_i(x)
- Project R^c onto col(D): z = (D D^+) R^c
- Facet condition: |c^T z| ≤ Σ_k |c^T D_k|

**Boundary curves** occur where equality holds (slack = 0):
```
slack_i(x) = Σ_k |c^T D_k(x)| - |c^T z(x)| = 0
```

## Initial Distribution Analysis

For initial state distribution α = [α, 1-α]:
```
V(π, α) = α·v(s=0) + (1-α)·v(s=1)
```

### Extreme Points

The extreme points (vertices) of the feasible value function set correspond to:
1. **Corner policies**: p_o ∈ {0,1} for all observations
2. **Boundary policies**: At least one constraint is tight (q_i = ±1)

### Optimal Policy Regions

For each α ∈ [0,1], we determine which extreme point maximizes V(π, α).

**Boundary equations**: Where two policies switch optimality:
```
α·v₁[0] + (1-α)·v₁[1] = α·v₂[0] + (1-α)·v₂[1]

Solving for α:
α* = (v₂[1] - v₁[1]) / ((v₁[0] - v₁[1]) - (v₂[0] - v₂[1]))
```

**Result**: 4 extreme points, **0 boundaries**
- Single policy dominates for all initial distributions
- Perfect observations eliminate competition between policies

### Configuration 2: Noisy Observations
```python
Beta = [[0.8, 0.2], [0.3, 0.7]]
R = [[1.0, 0.0], [0.0, 1.0]]
```

**Result**: 20 extreme points, **1 boundary**

**Boundary at α = 0.083607:**
- For α < 0.083607: Policy 1 optimal (p=[1,1], v=[5.73, 7.55])
- For α > 0.083607: Policy 0 optimal (p=[0,1], v=[7.61, 7.37])

**Derivation:**
```
Setting equal:
α·5.7273 + (1-α)·7.5455 = α·7.6051 + (1-α)·7.3741
α·(-2.0491) = -0.1713
α = 0.083607
```

### Configuration 3: Very Noisy Observations (MOST INTERESTING)
```python
Beta = [[0.65, 0.35], [0.35, 0.65]]
R = [[1.0, 0.0], [0.0, 1.0]]
```

**Result**: 25 extreme points, **2 boundaries**

This configuration exhibits **3 distinct optimal regions**:

#### Region 1: α < 0.575
- **Optimal Policy**: Policy 2 (p=[1,1])
- **Value**: v = [5.73, 7.55]
- **Interpretation**: Always take action 1 regardless of observation

#### Region 2: 0.575 < α < 0.740
- **Optimal Policy**: Policy 1 (p=[0,1])
- **Value**: v = [6.5, 6.5]
- **Interpretation**: Take action 1 only when observing o=1

#### Region 3: α > 0.740
- **Optimal Policy**: Policy 0 (p=[0,0])
- **Value**: v = [7.07, 4.89]
- **Interpretation**: Always take action 0 regardless of observation

**Boundary Equations:**

*Boundary 1 (α* = 0.575):*
```
Policy 2: v = [5.7273, 7.5455]
Policy 1: v = [6.5, 6.5]

α·5.7273 + (1-α)·7.5455 = α·6.5 + (1-α)·6.5
α·(-1.8182) = -1.0455
α = 0.575
```

*Boundary 2 (α* = 0.740):*
```
Policy 1: v = [6.5, 6.5]
Policy 0: v = [7.0652, 4.8913]

α·6.5 + (1-α)·6.5 = α·7.0652 + (1-α)·4.8913
α·(-2.1739) = -1.6087
α = 0.740
```

## Key Insights

### 1. Observation Noise Creates Structural Complexity
- **Perfect observations** (Beta = I): Single dominant policy
- **Noisy observations**: Multiple optimal regions emerge
- **Very noisy observations**: Richer structure with more boundaries

### 2. Geometric Interpretation
The feasible value function set forms a **convex polytope** in value space:
- Vertices correspond to extreme policies
- For each initial distribution α, we optimize a linear functional over this polytope
- Boundaries occur where the optimal vertex changes

### 3. Three-Region Structure in Very Noisy Case
- **Low α** (state 0 likely): Aggressive policy (always action 1)
- **Medium α**: Moderate policy (action depends on observation)
- **High α** (state 1 likely): Conservative policy (always action 0)

The observation noise creates ambiguity that allows different policies to be optimal depending on the prior over states.

### 4. Explicit Boundary Formulas
The boundaries have simple closed-form expressions:
```
α* = (v₂[1] - v₁[1]) / ((v₁[0] - v₁[1]) - (v₂[0] - v₂[1]))
```

This allows analytical characterization without numerical optimization.

## Mathematical Details

### Support Function Inequalities

For direction y ∈ ℝ^S, the support function condition is:
```
y^T R^c(x) ≤ Σ_k p^Δ_k |y^T (A^k x - b^k)|
```

By sampling many directions y (unit circle in 2D), we obtain linear inequalities that approximate the zonotope boundary.

### Exact Nonlinear Boundaries

The exact boundary occurs where q_i = ±1 for some i:
```
D(x) q = R^c(x), with q_i ∈ {-1, +1}
```

These are **nonlinear** constraints in x (since D and R^c depend on x).

### Rank Considerations

- **Full rank** D(x): System D(x)q = R^c(x) has unique solution
- **Rank deficient** D(x): Must check if R^c ∈ col(D) and feasibility of q ∈ [-1,1]^K

For most 2-state examples, D(x) has full rank on the boundary.
